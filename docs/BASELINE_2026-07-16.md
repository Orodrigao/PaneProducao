# Baseline de estabilidade — 2026-07-16

**Base avaliada:** `origin/main` no commit `cdf26f7`

**Objetivo:** estabelecer uma referência reproduzível antes de priorizar
regressões e retomar o hardening de segurança.

## Resultado executivo

O código parte de uma instalação limpa, passa em tipos, testes e build. O lint
também retorna sucesso, mas carrega 145 avisos. Todas as 31 páginas HTML
exportadas responderam localmente.

Esse resultado confirma a integridade técnica básica do artefato. Ele não
confirma login, permissões, acesso por perfil ou loja, consultas ao Supabase nem
interações do usuário.

## Verificações

| Verificação | Resultado |
| --- | --- |
| `npm ci` | passou |
| `npm run lint` | passou com 145 avisos |
| `npx tsc --noEmit` | passou |
| `npm test` | 19 arquivos e 109 testes passaram |
| `npm run build` | passou; export estático gerado |
| smoke HTTP do diretório `out/` | 31 de 31 páginas HTML responderam `200` |

Ambiente usado:

- Node.js `v24.13.0`;
- npm `11.6.2`;
- Next.js `15.5.18`.

## Dívida encontrada

### Lint

Os 145 avisos estão distribuídos em 15 arquivos:

- 106 usos de `no-explicit-any`;
- 35 símbolos não usados;
- 3 dependências incompletas de hooks;
- 1 aviso de fonte customizada.

O comando retornar sucesso não deve ser interpretado como ausência de dívida.
Essa limpeza deve ser separada por módulo para reduzir risco de regressão.

O próprio Next.js também avisa que `next lint` será removido no Next.js 16. A
migração para ESLint CLI deve ocorrer antes de uma atualização de versão.

### Dependências

`npm audit` registrou 3 alertas moderados, nenhum alto e nenhum crítico:

- `js-yaml`, transitivo;
- `postcss`, transitivo pelo Next.js;
- `next`, direto, relacionado à cadeia do `postcss`.

A correção automática sugerida pelo npm aponta para uma mudança principal
incompatível e até uma versão antiga do Next.js. Não executar `npm audit fix`
sem uma tarefa específica de atualização, análise do advisory e regressão
completa.

## Limitação do navegador

O controle interativo do navegador não estava disponível na sessão que produziu
este baseline. Por isso, não foram validados:

- login e recuperação de senha;
- autorização por perfil e loja;
- carregamento de dados reais;
- estados vazio, erro, sucesso e repetição de ação;
- fluxos críticos em viewport móvel.

O smoke HTTP é complementar e não substitui esses testes. Nenhuma credencial foi
usada e nenhuma escrita foi feita em produção.

## Prioridades resultantes

1. Executar smoke visual autenticado dos fluxos críticos em sessão com controle
   de navegador.
2. Reproduzir e priorizar bugs observados, sem misturá-los em uma correção
   genérica.
3. Tratar alertas de dependência em tarefa própria.
4. Reduzir avisos de lint em lotes por módulo.
5. Retomar auditoria live somente leitura de Auth/RLS.
