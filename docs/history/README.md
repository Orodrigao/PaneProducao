# Registros históricos

Tudo nesta pasta descreve um momento passado do projeto: auditorias,
resultados de aplicação, matrizes, planos e tarefas já encerradas.

Regras:

- nada aqui define o estado atual — para isso, `docs/CURRENT_STATE.md`;
- não copie "próximo passo", status, SQL ou imperativo destes arquivos sem
  nova auditoria no código e no banco;
- links e caminhos internos podem estar desatualizados; são preservados como
  evidência, não como referência viva;
- documento vivo em `docs/` que perder vigência vem para cá, sem edição.
